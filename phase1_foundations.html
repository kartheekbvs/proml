01. Setup & Environment Depth

WHAT IS THIS?

The Infrastructure Layer

In PHP, you likely had XAMPP or just installed brew install php. In Python, doing this is a recipe for disaster. Why? Because Machine Learning libraries are volatile. Library A needs Python 3.10; Library B needs Python 3.12. If you use one global Python, they will destroy each other.

Why Python is #1 for ML?

Python isn't fast (it's interpreted). However, it is the perfect "Glue." All heavy lifting for ML (Matrices, Neural Networks) is written in C++ and Fortran. Python allows us to call those fast libraries with high-level, human-readable syntax.

[ DEVELOPER ] Writes readable code in .py files | v [ INTERPRETER ] Translates line-by-line | v [ C-EXTENSIONS ] (NumPy/PyTorch) Heavy Math in C++ | v [ HARDWARE ] GPU / CPU crunching numbers

The Golden Stack: Pyenv + Venv + Pip

1

Pyenv (The Manager)

Analogy: Like a wardrobe manager. You can choose to wear the "3.12 suit" for one project and the "3.9 suit" for another.

# Install a specific version pyenv install 3.12.2 # Set it globally pyenv global 3.12.2

Virtual Environments (Venv)

Analogy: An island. Everything you install on the island stays on the island and doesn't pollute the mainland.

# Create the island python -m venv venv # Step onto the island (Mac/Linux) source venv/bin/activate # Step onto the island (Windows) .\venv\Scripts\activate

02. Core Syntax & Logic: The Indentation Shift

THE MENTAL MODEL

Python has no curly braces for blocks. This is a shock to PHP developers. You define scope by using indentation (whitespace). If you use 3 spaces instead of 4, the program crashes.

PHP (Explicit)

if ($x > 5) {
    echo "High";
}

Python (Implicit)

if x > 5:
    print("High")
# Indentation defines the block!

Common Indentation Errors

Beginners often mix Tabs and Spaces. Most modern IDEs (VS Code) convert tabs to 4 spaces automatically. Always stick to 4 spaces.

IndentationError: expected an indented block

03. Dynamic Typing vs Type Hints

Python is dynamically typed, meaning a variable can be an integer now and a string later. However, in 2025, professionals use Type Hints to make code readable and help FastAPI validate data.

typing_demo.py
# The Professional Way (Type Hints)
from typing import List, Optional

age: int = 25
name: str = "Ali"
is_active: bool = True

# A list that only holds floats
scores: List[float] = [98.5, 87.0, 92.1]

# A variable that could be a string OR None
user_id: Optional[str] = None

def process_ml_data(input_val: float) -> str:
    """Explicitly says: Input is float, Output is string."""
    return f"Processed: {input_val}"

05. Functions & Decorators Depth

Machine Learning frameworks like FastAPI depend heavily on Decorators. You must understand them to build modern APIs.

ANALOGY

A decorator is like a gift wrapper. The gift (your function) is still inside, but the wrapper (the decorator) can do something before or after the gift is opened.

decorators.py
from functools import wraps
import time

def logger_decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(f"--- Running: {func.__name__} ---")
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"--- Completed in {end - start:.4f}s ---")
        return result
    return wrapper

@logger_decorator
def expensive_ml_inference(data_point: float):
    time.sleep(1) # Simulating heavy work
    return data_point * 0.5

# Calling it
print(expensive_ml_inference(42.0))

Argument Expansion: *args and **kwargs

Sometimes you don't know how many arguments you will get. - *args: List of positional arguments. - **kwargs: Dictionary of keyword arguments.

06. Professional Object Oriented Python

PHP DIFFERENCE

Python has no true private properties. While PHP has private $var, Python just has _var (a pinky-promise not to touch it) and __var (name mangling).

The Master Class: Dataclasses

For ML, we often just need classes to hold data. dataclasses (Python 3.7+) automate the boring stuff like __init__.

oop_pro.py
from dataclasses import dataclass, field
from abc import ABC, abstractmethod

# 1. Abstract Base Class (Blueprint)
class AIModel(ABC):
    @abstractmethod
    def predict(self, x: float):
        pass

# 2. Dataclass for configuration
@dataclass
class ModelConfig:
    name: str
    tolerance: float = 0.001
    history: list = field(default_factory=list)

# 3. Concrete Implementation
class LinearModel(AIModel):
    def __init__(self, config: ModelConfig):
        self.config = config

    def predict(self, x: float):
        return x * 1.5

# Usage
cfg = ModelConfig("PrimaryBot")
bot = LinearModel(cfg)
print(bot.predict(10.0))

08. Mini-Project: The "Foundations" CLI Shell

We will build a simple CLI tool that manages "Predictions." It uses OOP, IO, File paths, and Lists.

shell.py
import json
from pathlib import Path
from dataclasses import dataclass, asdict
import sys

@dataclass
class Prediction:
    input_val: str
    output_val: float
    timestamp: str

class PredictionManager:
    def __init__(self, db_name="logs.json"):
        self.path = Path(db_name)
        self.logs = self._load()

    def _load(self):
        if not self.path.exists(): return []
        return [Prediction(**p) for p in json.loads(self.path.read_text())]

    def add(self, p: Prediction):
        self.logs.append(p)
        self._save()

    def _save(self):
        data = [asdict(p) for p in self.logs]
        self.path.write_text(json.dumps(data, indent=4))

    def show(self):
        for idx, entry in enumerate(self.logs):
            print(f"[{idx}] {entry.input_val} -> {entry.output_val}")

def main():
    manager = PredictionManager()
    
    while True:
        print("\n--- ML PREDICTION SHELL ---")
        print("1. Add Log | 2. Show Logs | 3. Exit")
        choice = input("> ")

        if choice == "1":
            inp = input("Input text: ")
            import random
            pred = Prediction(inp, random.random(), "2024-03")
            manager.add(pred)
        elif choice == "2":
            manager.show()
        elif choice == "3":
            sys.exit()

if __name__ == "__main__":
    main()

09. Quiz & Self-Check Mastery

1. Why is if __name__ == "__main__": used?
It prevents your script from running automatically when another file imports it. It's the "Entry Point" guard.

2. What is the difference between a Tuple and a List?
Tuples are Immutable (fixed size/values). Lists are Mutable. Tuples are faster and safer for constants.

3. How do you handle a missing key in a Dictionary?
Use my_dict.get("key", "default_value") instead of my_dict["key"] to avoid a KeyError crash.
