Phase 1: Python Foundations

Zero to comfortable. Move from PHP's world into the powerhouse of Machine Learning.

Prerequisites: Basic logic, know what a variable is in PHP.

Estimated Reading Time: 120 minutes

01. Why Python for ML?

Is Python slow? Yes, the core engine is slower than C++ or Java. Then why ML? Because Python acts as a "wrapper." The heavy math happens in C++ (inside libraries like NumPy), but we control it with easy, readable Python code.

In PHP: You build web pages. Focus is on Request/Response. Strong in templates.
In Python: You build models & logic. Focus is on data manipulation. Strong in math & AI.
[ Your Python Code ] <-- Easy to write | [ C++ ML Libraries ] <-- Fast execution | [ GPU / Processor ] <-- Hard work

02. Core Concept: Readability

Python's primary philosophy is: "Readability counts."

What is this? Python uses whitespace (indentation) instead of curly braces {} or semicolons ;. If your code isn't tidy, it won't run. This forces every developer to write clean code.

03. Mental Model: The Executive Assistant

If PHP is like a Web Concierge (handling guests/browsers), Python is like an Executive Assistant for a Data Scientist. It's smart, understands high-level instructions, and converts your messy notes (data) into clean reports (models).

04. Absolute Minimum Working Code

Every Python script starts with a simple .py file. No boilerplate needed.

hello.py
# Simple script to greet and calculate
name = "Ali"
age = 25
print(f"Hello {name}, in 5 years you will be {age + 5}")

# List of ML topics
topics = ["Foundations", "FastAPI", "Deployment"]

# Loop through (PHP's foreach)
for topic in topics:
    print(f"Learning: {topic}")

05. Line-by-Line Walkthrough

1 name = "Ali": Variables don't need $.
2 print(f"..."): This is an f-string. Like PHP's "{$var}" but cleaner.
3 for topic in topics:: Note the colon at the end. That means "start of block."

06. How to Run It

python hello.py

IndentationError? If you get this, check your spaces. Python strictly requires 4 spaces for every level of code inside a block.

09. Sub-Topic: Setup & Environment

Why not just install Python?

If you install it globally, one project might need version 3.10 and another needs 3.12. They will fight. We use Virtual Environments to keep them separate.

# Create a private folder for libraries python -m venv venv # "Activate" it (Windows) .\venv\Scripts\activate # Install a library (Like Composer) pip install python-dotenv

Tip: Always look for the (venv) prefix in your terminal to know you are "inside" your project bubble.

10. Sub-Topic: Core Syntax & Types

Python has Dynamic Typing. This means a variable can hold a string now and a number later (just like PHP).

syntax.py
# Variables
x = 10       # integer
y = 10.5     # float
is_ml_pro = True # boolean (starts with Capital!)

# Checks type (PHP's gettype())
print(type(x))

# Multi-line string (Triple quotes)
query = """
SELECT * 
FROM users 
WHERE age > 18
"""

None: This is Python's null. Use it when a variable has no value yet.

11. Sub-Topic: Data Structures

Lists (PHP Indexed Array)

python
items = ["a", "b", "c"]

Dictionaries (PHP Associative Array)

python
user = {"name": "Ali", "role": "admin"}
print(user["name"])
print(user.get("age", "N/A")) # Safer, won't crash if 'age' missing

List Comprehensions (Pure Python Magic)

Convert lists in one line. It replaces long foreach loops.

python
numbers = [1, 2, 3, 4, 5]
# Get only squares of even numbers
evens_squared = [n * n for n in numbers if n % 2 == 0]
# result: [4, 16]

def add_numbers(a: int, b: int) -> int:

    return a + b

result = add_numbers(b=10, a=5) # Keyword order doesn't matter

13. Sub-Topic: OOP & Dataclasses

Python doesn't use public/private keywords. It uses underscores: _name means "don't touch me directly."

oop.py
from dataclasses import dataclass

@dataclass
class ModelInfo:
    name: str
    version: float
    is_ready: bool = False

# Automatic constructor!
m = ModelInfo(name="SentimentBot", version=1.2)
print(m)

14. Sub-Topic: Modules & Files

Pathlib: The Modern Way

Forget messy string paths. Use the Path object.

python
from pathlib import Path

# Create a path regardless of Windows/Mac
data_file = Path("data") / "results.csv"

# Error Handling (Try/Catch)
try:
    with open(data_file, "r") as f:
        print(f.read())
except FileNotFoundError:
    print("Oops! File is missing.")

N-3. Mini Project: ML Model Registry CLI

Let's build a small tool that stores and lists ML models in a JSON file.

registry.py
import json
from pathlib import Path
from dataclasses import dataclass, asdict

# 1. Define the Data Structure
@dataclass
class MLModel:
    name: str
    accuracy: float
    framework: str

# 2. Storage Setup
STORAGE_FILE = Path("models.json")

def save_model(model: MLModel):
    # Load existing or empty list
    if STORAGE_FILE.exists():
        models = json.loads(STORAGE_FILE.read_text())
    else:
        models = []
    
    # Add new model (convert dataclass to dict)
    models.append(asdict(model))
    STORAGE_FILE.write_text(json.dumps(models, indent=4))
    print(f"✅ Model '{model.name}' saved!")

def list_models():
    if not STORAGE_FILE.exists():
        print("No models found.")
        return
    
    models = json.loads(STORAGE_FILE.read_text())
    print("\n--- Registered Models ---")
    for m in models:
        print(f"[{m['framework']}] {m['name']} - Acc: {m['accuracy']*100}%")

# 3. Simple CLI Interface
if __name__ == "__main__":
    # Test adding a model
    new_m = MLModel(name="CatDetector", accuracy=0.98, framework="PyTorch")
    save_model(new_m)
    list_models()

N-2. Quiz & Interview Questions

1. Why is __init__ used in classes?
It's the constructor. It runs as soon as you create an object. (Like __construct in PHP).

2. What is a "Mutable" object?
Something that can change. Lists are mutable. Tuples are NOT (Immutable).

3. What happens if you miss an indent?
Python raises an IndentationError and the script crashes immediately.
