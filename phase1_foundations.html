MODULE 1.1

Setup & Environment

What is this? Python setup is about creating a clean, isolated workspace for your project. Think of it like a chef having a dedicated cutting board for raw meat and another for vegetables. We don't want library versions from one project messing up another.

Virtual Environments (The Secret Sauce)

In PHP, you might have global extensions. In Python, we use venv.

Terminal
# Create the environment
python -m venv venv

# Activate it
source venv/bin/activate  # Mac/Linux
.\venv\Scripts\activate   # Windows

Environment Variables (.env)

Never hardcode API keys. Use python-dotenv.

.env
API_KEY=your_secret_key_here
DEBUG=True
MODULE 1.2

Core Syntax (vs PHP)

Python indentation: Unlike PHP which uses {}, Python uses 4 spaces. It's not just for style; the code won't run if it's wrong!

python
# Variable declaration (no $)
name = "Ali"
age = 25

# F-Strings (Modern String Formatting)
print(f"Hello {name}, you are {age} years old.")

# Type Hints (Crucial for FastAPI)
def greet(name: str) -> str:
    return f"Hello {name}"
MODULE 1.3

Data Structures

Lists and Dictionaries are your bread and butter.

  • Lists: Ordered sequences (PHP Indexed Arrays).
  • Dictionaries: Key-Value pairs (PHP Associative Arrays).
python
# List Comprehension (Python's superpower)
nums = [1, 2, 3, 4]
squares = [n*n for n in nums if n > 2] # [9, 16]

# Dictionary
user = {"id": 1, "name": "Ali"}
print(user.get("age", 0)) # Safer than user["age"]
MODULE 1.4

Functions & Decorators

Decorators: These are functions that "wrap" other functions to change their behavior. FastAPI uses these for routes (e.g., @app.get("/")).

python
def logger(func):
    def wrapper(*args, **kwargs):
        print(f"Calling {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

@logger
def add(a, b):
    return a + b
MODULE 1.5

OOP: Classes and Data

python
from dataclasses import dataclass

@dataclass
class User:
    id: int
    name: str

# No need for manual __init__!
new_user = User(id=1, name="Ali")
